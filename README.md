# Юнит-Экономика PRO

**Калькулятор реальной прибыльности товара** — от закупки в Китае (или производства в РФ) до чистой прибыли после налогов на маркетплейсах.

Считает полную экономику **одной единицы товара** для **ООО на ОСНО**: себестоимость, логистику, таможню, расходы маркетплейсов (Wildberries, Ozon, Яндекс Маркет, Мегамаркет), НДС и налог на прибыль, а также 16 финансовых метрик.

> Чистый HTML/CSS/JavaScript. Без сервера, без сторонних библиотек, **работает офлайн**. Данные хранятся локально (IndexedDB). Устанавливается как **PWA-приложение** на телефон и ПК.

## ✨ Возможности

- **18 блоков расчёта**: закупка → характеристики → логистика по Китаю → международная логистика → таможня → производство РФ → акриловые краски → расходы ООО → распределение накладных → маркетплейсы → налоги → аналитика → сценарии → сравнение → визуализация → отчёты → сохранение → дашборд собственника.
- **Модульность**: включайте направления под проект — импорт из Китая, контрактное производство РФ, акриловые краски, готовая продукция РФ.
- **Мультивалютность**: закупка в CNY / USD / EUR, отчёты в рублях, курс вводится вручную.
- **Маркетплейсы**: Wildberries, Ozon, Яндекс Маркет, Мегамаркет; модели **FBO / FBS / realFBS**; учёт комиссии, логистики, хранения, приёмки, эквайринга, рекламы/ДРР, акций, скидок, **процента выкупа и возвратов** (логистика туда-обратно), утилизации, брака и потерь.
- **Налоги ОСНО**: входящий и исходящий НДС, НДС к уплате, налог на прибыль 25%. **НДС не включается в себестоимость** автоматически — входящий принимается к вычету.
- **16 метрик**: себестоимость единицы/партии, валовая/маржинальная/чистая прибыль, рентабельность, ROI, ROMI, точка безубыточности, срок окупаемости, прибыль за месяц/год, минимальная и рекомендуемая цена, максимальная скидка без убытка.
- **Сценарии**: пессимистичный / базовый / оптимистичный (множители цены, объёма, выкупа, стоимости закупки и логистики).
- **Сравнение** до 20 товаров по прибыли, ROI, окупаемости и рентабельности.
- **Графики** (SVG, без библиотек): структура себестоимости, расходы, безубыточность, окупаемость, прибыль по сценариям и по маркетплейсам.
- **Экспорт**: Excel (.xls), CSV, PDF (печать браузера), JSON (перенос проекта).
- **Дашборд собственника**: вложенные средства, остаток оборотного капитала, прибыль месяца/года, средняя маржинальность, лучший и худший товар.
- **Проекты**: создание, сохранение, **автосохранение каждые 30 секунд**, загрузка, дублирование, удаление, экспорт/импорт JSON. У каждого проекта уникальный ID, имя, даты создания и изменения.
- **Темы**: тёмная и светлая. **Адаптив** под ПК и смартфон.

## 🚀 Публикация на GitHub Pages

### Способ A. Через сайт GitHub (без программ, 5 минут)

1. Зайдите на **https://github.com** и войдите в аккаунт (или зарегистрируйтесь).
2. Нажмите **New** (зелёная кнопка) → создайте репозиторий, например `unit-economика` или `profit-calc`.
   - Видимость: **Public** (для бесплатного GitHub Pages).
   - Галочку «Add a README» можно **не** ставить — README уже есть в проекте.
3. На странице пустого репозитория нажмите **uploading an existing file**.
4. Перетащите в окно **все файлы и папку `icons`** из этого проекта:
   ```
   index.html
   style.css
   script.js
   manifest.json
   sw.js
   favicon.svg
   .nojekyll
   README.md
   icons/        (если генерировали PNG — необязательно)
   ```
   > Важно: загрузите именно **содержимое** папки проекта, чтобы `index.html` лежал в корне репозитория.
5. Внизу нажмите **Commit changes**.
6. Откройте **Settings** репозитория → слева **Pages**.
7. В разделе **Build and deployment** → **Source** выберите **Deploy from a branch**.
8. **Branch**: выберите `main`, папка `/ (root)` → **Save**.
9. Подождите 1–2 минуты. Вверху страницы Pages появится ссылка вида:
   ```
   https://ВАШ-ЛОГИН.github.io/НАЗВАНИЕ-РЕПОЗИТОРИЯ/
   ```
   Откройте её — приложение готово.

### Способ B. Через Git (командная строка)

```bash
cd путь/к/папке/проекта
git init
git add .
git commit -m "Юнит-Экономика PRO"
git branch -M main
git remote add origin https://github.com/ВАШ-ЛОГИН/НАЗВАНИЕ.git
git push -u origin main
```
Затем выполните шаги **6–9** из способа A (Settings → Pages → Deploy from a branch → main → /root).

> **Файл `.nojekyll`** в корне обязателен — он отключает обработку Jekyll, чтобы GitHub Pages отдавал файлы как есть.

## 📱 Установка как приложение (PWA)

После открытия сайта по ссылке `github.io`:

- **Android (Chrome)**: меню ⋮ → **«Установить приложение»** / «Добавить на главный экран».
- **iPhone/iPad (Safari)**: кнопка **Поделиться** → **«На экран Домой»**.
- **Windows/macOS (Chrome/Edge)**: значок установки ⊕ в адресной строке → **«Установить»**.

Приложение запускается в отдельном окне и **работает без интернета** (после первого открытия кэшируется сервис-воркером).

## 💾 Хранение данных

- **Проекты** — в **IndexedDB** браузера (локально на устройстве). Если IndexedDB недоступен, используется LocalStorage как резерв.
- **Настройки интерфейса** (тема, выбранный раздел, оборотный капитал) — в **LocalStorage**.
- Данные **никуда не передаются** и хранятся только в текущем браузере. Для резервной копии и переноса используйте **Экспорт JSON** (раздел «Отчёты»).

## 🧮 Методология расчёта (кратко)

- **Расчётный (тарифицируемый) вес** международной логистики: `max(вес брутто; объёмный вес)`. Для авиа объёмный вес = объём ÷ делитель (по умолчанию 6000).
- **Таможенная стоимость**: по **CIF** (товар + доставка + страховка до границы ЕАЭС).
- **Пошлина** входит в себестоимость; **таможенный НДС** — входящий НДС к вычету (в себестоимость не входит).
- **Входящий НДС** с услуг (логистика МП, комиссия, брокер, производство РФ) принимается к вычету; в себестоимость идёт сумма **без НДС**.
- **Постоянные расходы** распределяются на единицу по **плану продаж в месяц** (или в год — переключается).
- **Выкуп и возвраты**: прямая логистика начисляется на все отправления (с учётом % выкупа), обратная — на возвраты; учитываются утилизация, брак и потери.
- **Налог на прибыль** считается с операционной прибыли (ставка 25% по умолчанию).

> Расчёты предназначены для управленческого учёта и оценки юнит-экономики. Итоговые ставки пошлин, тарифы и комиссии уточняйте в актуальных источниках (ТН ВЭД, тарифы перевозчиков и маркетплейсов).

## 📂 Структура проекта

```
index.html        — разметка и оболочка приложения
style.css         — стили, тёмная/светлая темы, адаптив, печать
script.js         — движок расчётов, IndexedDB, графики, экспорт, PWA
manifest.json     — манифест PWA (имя, иконки, цвета)
sw.js             — сервис-воркер (офлайн-кэш)
favicon.svg       — векторная иконка (используется и как иконка PWA)
.nojekyll         — отключает Jekyll на GitHub Pages
make-icons.ps1    — (необязательно) генерация PNG-иконок для максимальной совместимости
README.md         — этот файл
```

## 🖼️ Опционально: PNG-иконки

SVG-иконки достаточно для установки PWA в современных браузерах. Если нужны PNG (например, для старых устройств или iOS), на Windows выполните в PowerShell в папке проекта:

```powershell
powershell -ExecutionPolicy Bypass -File make-icons.ps1
```

Скрипт создаст `icons/icon-192.png`, `icons/icon-512.png`, `icons/icon-maskable-512.png` и `icons/apple-touch-icon.png`. После этого добавьте их в массив `icons` файла `manifest.json`.
